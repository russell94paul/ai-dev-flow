# ai-dev-flow

A structured AI-driven development workflow system.

---

## 🚀 What is this?

**ai-dev-flow** transforms software development from ad-hoc prompting into a **repeatable, structured workflow**.

Instead of:

> “Ask AI to build something and hope for the best”

You get:

> **idea → feature → PRD → plan → implementation → artifacts**

---

## 🧠 Why this exists

Most AI-assisted development today is:

* unstructured
* inconsistent
* hard to debug
* difficult to reproduce

ai-dev-flow introduces:

* ✅ structured thinking
* ✅ repeatable workflows
* ✅ persistent knowledge (Obsidian-style)
* ✅ higher quality outputs

---

## ⚙️ Core Workflows

### 🟢 New Project

Break a project idea into structured features:

```bash
ai new-project "AI trading coach"
```

---

### 🟣 Feature Workflow

Run the full development lifecycle:

```bash
ai feature "Build API sync feature"
```

This executes:

1. **Grill** → clarify requirements
2. **PRD** → define the feature
3. **Plan** → break into steps
4. **TDD** → implement safely

---

## 📂 Output (Artifacts)

Each feature produces structured outputs:

```
feature-name/
  grill.md
  prd.md
  plan.md
  tdd.md
```

These can be stored in an Obsidian vault to create a **persistent developer knowledge base**.


Ensure:

* Claude CLI is installed
* AutoHotkey (Windows) is configured
